- **Re-attempt multi-commit-skip `ignoreCommand` fix with out-of-prod test infra.** The original `"ignoreCommand": "git diff --quiet HEAD^ HEAD -- ."` in both `windex-admin/vercel.json` and `windex-expo/vercel.json` (introduced 2026-05-11) silently skips builds when a multi-commit push lands the meaningful change anywhere but the tip commit. `HEAD^..HEAD` only inspects the single commit at the push's tip; squash-merges are immune (always single-tip), but rebase-merges of mixed-scope PRs are vulnerable. Smoking gun was PR #17 (2026-05-20): a rebase-merge landing migration + windex-admin UI + root docs as four commits, HEAD was a root-level docs commit, HEAD^ was another root-level docs commit, neither touched `windex-admin/`, build was skipped, the new `/cup-champions` UI shipped to master but never deployed. **Three fix attempts on 2026-05-20 / 2026-05-21 all failed in production** (full trail commits `505a7ac`, `ff72f2f`, `bd98dd4`, reverted by the commit at the time of this BACKLOG entry): (1) **Plain `VERCEL_GIT_PREVIOUS_SHA` diff** — failed with `fatal: bad object 3aaba812…` because the previous-deploy SHA wasn't in Vercel's shallow clone (depth ~10). windex-expo got lucky (recent enough SHA); windex-admin didn't (SHA from May 19, way outside the shallow window). - **Replace date-derived "current season" with an explicit flag.** The `championship_results_check_membership` trigger in migration 032 derives "current" from `seasons.start_date <= today AND seasons.end_date >= today` (today = `now() AT TIME ZONE 'UTC'`). - **windex-admin admin session is access-token-only (1h, no refresh) — saves hard-fail at JWT expiry.** Login (`windex-admin/src/pages/Login.tsx` `handleEmailLogin`) does the password grant against `/auth/v1/token?grant_type=password` but stores ONLY `data.access_token` (line ~51) and discards `refresh_token`; the token lives in `sessionStorage` (`late_add_admin_jwt`, `windex-admin/src/api/client.ts`). There is no `supabase-js` client and no `autoRefreshToken` — every API module raw-fetches with the stored token. With the Supabase default 1-hour access-token TTL and no refresh path, any admin write after the hour 401s (PostgREST code `PGRST301`, "JWT expired"). Surfaced 2026-06-17: `EditChampionshipResultsModal` save died with "Failed to clear existing results: JWT expired". Real fix (deferred): persist `refresh_token` at login + add a refresh mechanism (or adopt a `supabase-js` client with `autoRefreshToken`), and add a shared 401/`PGRST301` interceptor so every raw-fetch module (championshipResults, and any other module not routed through `client.ts apiFetch`) recovers uniformly instead of each hard-failing.
